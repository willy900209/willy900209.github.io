碩士論文：日冕洞演化對太陽風的影響

本研究以 SDO/AIA 193 Å 太陽影像與 OMNI 每小時太陽風速度資料為基礎， 建立日冕洞特徵擷取與太陽風速度預測流程。

研究流程首先對太陽影像進行前處理，包括亮度轉換、太陽盤面遮罩、 邊緣亮度修正與雜訊排除。接著使用 K-Means 分群、形態學處理與面積篩選方法， 從太陽盤面中擷取日冕洞候選區域，並計算其面積、亮度、形態、空間位置與時間演化特徵。

在模型建構方面，本研究以 XGBoost 建立雙模型架構，分別捕捉整體太陽風速度趨勢與高速太陽風峰值變化。 最後透過動態融合與傳播延遲修正機制，改善太陽端日冕洞特徵與地球端太陽風觀測之間的時間對齊問題， 並使用 SHAP 分析模型預測結果與重要特徵。

Python / OpenCV / XGBoost / SHAP / Time Series / Feature Engineering

學術期刊：An Adaptive Image-Processing Pipeline for Coronal-Hole Candidate Extraction from Full-Disk SDO/AIA 193 Å Images

本專案為碩士論文延伸之影像處理研究，主題為「以自適應影像處理流程擷取 SDO/AIA 193 Å 太陽全盤面影像中的日冕洞候選區域」。日冕洞是高速太陽風的重要來源之一， 因此能否穩定且可重現地從太陽影像中擷取日冕洞結構，對後續太陽風分析與太空天氣研究具有重要意義。

本專案以 SDO/AIA 193 Å 太陽全盤面影像作為資料來源，設計一套完整的日冕洞候選區域擷取流程。 首先透過 Asinh 非線性亮度轉換強化低亮度區域，使日冕洞等暗區結構更加清楚；接著利用 FITS 標頭資訊建立動態太陽盤面遮罩，排除盤面外背景干擾。為降低太陽盤面徑向亮度差異造成的分割不穩定， 本專案進一步建立徑向背景亮度校正方法，再搭配 K-Means 自適應閾值分割擷取初步候選區域。

在後處理階段，本專案結合混合形態學處理與連通區域篩選機制，透過核心重疊比例、區域面積與鄰近距離等條件， 保留主要且結構連續的低亮度區域，同時移除零散雜訊與不穩定碎片。相較於傳統 Otsu 閾值法、 灰階累積閾值法與標準 K-Means 分割，本方法更重視日冕洞區域的結構完整性與結果穩定性。

此專案展現我在影像前處理、二值化分割、形態學運算、連通區域分析與科學資料處理上的實作能力， 也呈現我能將統計與程式能力應用於跨領域太陽影像資料分析，並建立可重現的自動化處理流程。

Python / OpenCV / Image Processing / K-Means Segmentation / Morphology / Solar Image Analysis

114 年國道智慧交通管理創意競賽

本專案參與交通部舉辦之「114 年國道智慧交通管理創意競賽」，主要目標為提升國道交通狀態預測之準確性。 除使用主辦單位提供之交通資料外，亦額外蒐集外部資料，以補足影響交通流量與壅塞狀況之相關因素。

針對不同資料特性，本專案進行適當的資料前處理，包括缺失值填補、異常值與錯誤數值排除等步驟， 確保後續模型訓練資料之品質與穩定性。

在模型建構方面，將處理後之資料導入機器學習模型進行預測，並結合短期趨勢判別模型， 進一步修正與強化最終預測結果。

為提升作品完整性與實用性，本專案亦設計使用者互動式介面，讓使用者能夠更直觀地查看預測結果與交通資訊， 並建立資料即時更新機制，使系統能反映最新交通狀態。

Industrial Defect Detection using Object Detection and Segmentation

本專案以 Kolektor Surface-Defect Dataset 2 作為工業表面瑕疵偵測資料集， 建立 YOLOv8-based industrial defect detection framework，並從 image-level screening、 object-level detection 與 pixel-level segmentation 三個層次評估模型表現。

在資料處理上，本專案將原始 bitmap mask 標註轉換為 YOLO detection 與 YOLO segmentation 格式， 使模型能同時學習 bounding box localization 與 pixel-level mask prediction。 由於資料集中 normal images 遠多於 defect images，因此進一步採用 normal-sample downsampling， 並建立 patch-based segmentation dataset，以改善小面積、低對比與不規則瑕疵在 segmentation 任務中的學習效果。

實驗結果顯示，normal = 600 的 YOLOv8n detection model 在 object-level test set 上達到 mAP@0.5 = 0.767 與 mAP@0.5:0.95 = 0.402。進一步將 detection confidence score 轉換為 image-level defect score 後，模型於 defect screening 任務上達到 F1-score = 0.852、 ROC-AUC = 0.960 與 PR-AUC = 0.904。

此專案展現我在 computer vision、YOLO object detection、segmentation label conversion、 class imbalance handling、ablation study 與模型評估上的實作能力，也呈現我能將深度學習方法 應用於工業品檢與瑕疵定位任務。

CNN-Based Image Deblurring with Residual Learning

本專案以影像去模糊為主題，建立基於卷積神經網路（CNN）的影像復原模型。 模型採用 residual learning 架構，使網路不直接重建完整清晰影像，而是學習模糊影像與清晰影像之間的殘差資訊， 再將預測殘差加回原始模糊影像以產生去模糊結果。

在模型設計方面，本專案由 Original CNN 逐步改良至 V2、V2 Ultimate 與 V2 Ultimate-FT。 V2 Ultimate 加入 enhanced residual blocks、channel attention、residual scaling、EMA、資料增強與 cosine scheduler； 最終再以 fine-tuning 方式建立 V2 Ultimate-FT 作為最終模型。

實驗結果顯示，最終模型在 Set5 測試集上將 PSNR 由 21.55 dB 提升至 27.20 dB， SSIM 由 0.878 提升至 0.980；在 Set14 測試集上，PSNR 由 20.57 dB 提升至 24.91 dB， SSIM 由 0.881 提升至 0.961。本專案亦補充 ablation study、推論速度分析、訓練曲線與困難案例分析， 展現我在深度學習模型設計、電腦視覺實驗分析與結果解釋上的能力。

Python / PyTorch / OpenCV / CNN / Residual Learning / Image Deblurring / Computer Vision

Document Corner Detection and Perspective Correction using OpenCV and CNN

本專案以文件影像角點偵測與透視校正為主題，建立一套從文件四角點定位、 透視轉換到影像增強的完整流程。專案目標是將手機拍攝中常見的傾斜文件、 透視變形文件與背景干擾影像，轉換為接近正面掃描的文件結果。

在方法設計上，本專案先使用資料集提供的 ground truth 四角點建立理想透視校正基準， 接著建立基於 OpenCV 的 contour-based baseline，包含灰階轉換、CLAHE 對比增強、 Canny edge detection、morphological operation、contour detection 與四邊形篩選。 此方法可作為傳統影像處理基準，用於比較深度學習模型的改善幅度。

在深度學習模型方面，本專案建立小型 CNN 與 pretrained ResNet18 corner regression model， 直接預測文件左上、右上、右下與左下四個角點座標。為提升模型對文件邊界與角點細節的辨識能力， ResNet18 模型採用 384 × 384 輸入解析度、Smooth L1 loss、AdamW optimizer， 並加入資料增強以提升模型穩定性。

實驗結果顯示，OpenCV baseline 在複雜背景與邊界不完整影像中穩定性不足； 小型 CNN 能明顯降低角點誤差，而 ResNet18 進一步提升整體表現。 後續加入 MIDV-500 與 SmartDoc 2015 Challenge 1 進行外部資料擴充後， ResNet18 V2 在混合 test set 上達到 mean corner error = 41.20 pixels， median error = 18.14 pixels，顯示外部資料擴充對文件角點定位具有明顯幫助。

此專案展現我在 OpenCV 影像處理、CNN/ResNet 模型訓練、角點座標回歸、 透視幾何校正、資料擴充、模型評估與 worst-case analysis 上的實作能力， 也呈現我能將傳統影像處理方法與深度學習模型整合成完整的文件影像前處理系統。

Taiwan Real Estate Data Cleaning and Power BI Dashboard

本專案以台灣實價登錄買賣資料為資料來源，建立一套 PostgreSQL-based data cleaning pipeline， 將原始 CSV 資料匯入資料庫後，透過 SQL 完成資料清洗、資料品質檢查、分析用資料表建立， 並串接 Power BI 製作視覺化儀表板。

在資料處理流程上，本專案採用分層資料表設計，依序建立 raw table、clean table、validated table 與 analysis-ready table。 raw table 用於保留原始匯入資料；clean table 負責文字清理、空字串轉換與資料型態轉換； validated table 則透過 SQL validation flags 標記缺失值、無效價格、無效建物面積與極端單價等資料品質問題； 最後建立 analysis-ready table 作為後續分析與視覺化的主要資料來源。

在 SQL 實作上，本專案使用 TRIM、NULLIF、CASE WHEN、regular expression、 type casting、GROUP BY 與 CREATE VIEW 等語法，完成資料清洗、數值欄位轉換、 資料品質檢查與 summary views 建立。建立的 views 包含行政區房價彙總、 建物型態房價彙總與資料品質問題統計，並作為 Power BI dashboard 的資料來源。

在視覺化方面，本專案將 PostgreSQL 資料表與 views 串接至 Power BI， 建立地區房價總覽、行政區平均單價與交易量關係、不同建物型態比較、 建物面積與交易總價關係，以及資料品質問題統計等圖表。 此專案展現我在 SQL 資料清洗、資料庫流程設計、資料品質控管與 BI 視覺化整合上的實作能力。
